4Order lifecycle

End-to-end event choreography for one order — happy path plus failure branches. Listeners are idempotent (gated by the ProcessedEvent table per service), so duplicate deliveries are safe.

sequenceDiagram autonumber actor C as Customer (web) participant GW as api-gateway participant OS as order-service participant MQ as minimonk.events participant WS as warehouse-service participant PS as payment-service C->>GW: POST /api/orders (JWT) GW->>OS: POST /orders OS->>OS: persist CustomerOrder (PENDING) OS-->>MQ: order.created OS-->>C: 201 {orderId} MQ->>WS: order.created alt Stock available WS->>WS: atomic decrement (SQL) +
insert StockReservation WS-->>MQ: stock.reserved MQ->>PS: stock.reserved MQ->>OS: stock.reserved (status → RESERVED) OS-->>MQ: payment.requested Note over PS: deterministic by card # alt Card 4242… PS-->>MQ: payment.succeeded MQ->>OS: status → READY_FOR_PICKING OS-->>MQ: order.ready_for_picking else Card 4000…0002 PS-->>MQ: payment.failed MQ->>OS: status → CANCELLED MQ->>WS: payment.failed → release reservation WS-->>MQ: stock.released MQ->>OS: order.cancelled end else Out of stock WS-->>MQ: stock.reservation.failed MQ->>OS: status → CANCELLED OS-->>MQ: order.cancelled end

One-liner summary: order.created → stock.reserved | stock.reservation.failed → payment.requested → payment.succeeded | payment.failed → order.ready_for_picking | order.cancelled (+ stock.released)

6Cross-cutting patterns

Load-bearing, not optional. Touching any of these requires care.

PatternWhere it livesWhat it guarantees
JWT everywhere libs/common-security · each service's SecurityConfig A bypassed/compromised gateway cannot grant unauthorized access. Endpoint-level role checks per service.
Idempotent handlers ProcessedEvent + ProcessedEventRepository in order & warehouse services Duplicate event deliveries do not double-transition orders or double-release stock.
Durable queues + DLQ RabbitConfig in each consumer service Messages survive broker restarts. Poison messages route to *.dead-letter instead of looping.
Event envelope RabbitEventPublisher (libs/common-events) wraps payloads in EventEnvelope with a UUID Stable dedup ID powering the idempotency check above.
Atomic stock decrement SQL-level in warehouse-service · contract pinned by ProductRepositoryTest No oversells under concurrent reservations.
DTO projection for order list OrderOverviewDto in OrderService Avoids N+1 lazy-loading items per row when listing orders.
